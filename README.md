# Gang of Four (GOF) Design Patterns

The Gang of Four Design Patterns is a set of solutions to common problems we encounter in software design and development. The term comes from the authors of the famous book “Design Patterns: Elements of Reusable Object-Oriented Software” by Erich Gamma, Richard Helm, Ralph Johnson, and John Vlissides, who are collectively known as Gang of Four.

GoF introduced 23 classic design patterns that help solve common problems in object-oriented software.
These patterns provide solutions to common design problems and help make software systems more modular, flexible and maintainable.

## Types of GOF Design Patterns

These patterns are divided into three categories:

### Creational: 
Deal with object creation (e.g., Singleton, Factory, Builder).

### Structural: 
Deal with composition of classes/objects (e.g., Adapter, Decorator, Facade).

### Behavioral:
Deal with object interaction and communication (e.g., Observer, Strategy).
